# Module 14 - IP (IPv4)

**Files:** `src/protocols/ip.c`, `src/protocols/ip.h`  
**Status:** Partial, with clean output boundary and generic L4 dispatch started  
**Depends on:** packet, interface, device, topology, ethernet, arp_cache, arp, simulator

---

## Purpose

The IPv4 module sits between Ethernet and transport/control protocols such as
ICMP, UDP, and TCP.

It has two jobs:

1. Receive IPv4 packets after Ethernet strips the Ethernet header.
2. Send IPv4 packets by building an IPv4 header and passing the packet back to
   Ethernet.
3. Demultiplex valid local IPv4 payloads to registered layer-4 handlers by
   protocol number.

The clean design has two send boundaries:

- `ip_output(...)` is the normal L3 API. Callers provide source IP,
  destination IP, protocol number, and payload. IP chooses the outgoing
  interface and resolves the L2 destination.
- `ip_send(...)` is the lower-level on-link helper. Its caller already knows
  the outgoing interface and destination MAC address.

Upper-layer protocols such as ICMP, UDP, and TCP should call
`ip_output(...)`. They should not be responsible for route lookup or ARP
lookup.

---

## Current Implementation Scope

Implemented now:

- `ip_init` initializes one `IpStack` and registers IP as the receive handler
  for every interface in `sim->topo`.
- `ip_stack_init` initializes an explicit IP stack object.
- `ip_stack_bind_interface` binds one interface to one IP stack.
- `ip_receive` validates a minimum IPv4 header, strips it, updates interface
  receive counters, exposes the L4 payload, and dispatches it through the
  bound stack's protocol-handler table when a handler is registered.
- `ip_stack_register_protocol` installs a handler/context pair for an IPv4
  protocol number on one IP stack.
- `ip_stack_unregister_protocol` removes a handler/context pair from one IP
  stack.
- `ip_send` builds a 20-byte IPv4 header and sends through Ethernet when the
  outgoing interface and destination MAC are already known.
- `ip_output` finds a source interface, checks direct-subnet reachability,
  looks up the destination IP in that interface's ARP cache, and then calls
  `ip_send`.
- `ip_checksum` computes the IPv4 header checksum.
- `ip_is_local` checks whether an IP address matches one interface on a
  `Device`.

Not implemented yet:

- Router longest-prefix route lookup.
- Default gateway / next-hop selection.
- Packet forwarding with TTL decrement.
- Fragmentation.
- ICMP error generation from IP receive/forwarding paths.

This is intentional. The module now has the right API shape, but the full
router behavior still depends on route-table and ARP-pending infrastructure.

## IPv4 Header

This project supports the minimum IPv4 header only: 20 bytes, no options.

```c
typedef struct __attribute__((packed)) IpHeader {
    uint8_t  version_ihl;
    uint8_t  dscp_ecn;
    uint16_t total_length;
    uint16_t identification;
    uint16_t flags_fragment_offset;
    uint8_t  ttl;
    uint8_t  protocol;
    uint16_t header_checksum;
    uint32_t src_ip;
    uint32_t dst_ip;
} IpHeader;
```

Field rules:

| Field | Current rule |
|---|---|
| `version_ihl` | Must contain IPv4 version in the high nibble. Send uses `0x45`. |
| `total_length` | Send writes `IP_HDR_LEN + payload->len` in network byte order. |
| `identification` | Send currently writes `0`. |
| `flags_fragment_offset` | Send currently writes `0`; fragmentation is not implemented. |
| `ttl` | Send uses `IP_DEFAULT_TTL`; receive rejects `ttl == 0`. |
| `protocol` | Caller supplied, for example `IPPROTO_ICMP`. |
| `header_checksum` | Zeroed before checksum calculation, then filled. |
| `src_ip`, `dst_ip` | Stored in network byte order in the header. |

Important byte-order convention:

- Interface addresses, ARP packet addresses, and IPv4 header addresses are
  stored in network byte order.
- Public `ip_send` and `ip_output` parameters `src_ip` and `dst_ip` are host
  order. These functions convert them before writing wire headers or comparing
  against interface addresses.
- `arp_cache_lookup` currently expects the IP address in host order because
  `arp_cache_add` stores `ns_ntohl(sender_protocol_addr)`.
- `ip.c` uses two ARP-related modules for output:
  `arp_cache.h` provides lookup and pending-packet queue APIs, while `arp.h`
  provides ARP request transmission. Any build or KLEVA target for `ip.c` must
  link both `src/protocols/arp_cache.c` and `src/protocols/arp.c`. The ARP
  cache contract is specified in [10_arp_cache.md](10_arp_cache.md).

## Receive Path

Ethernet receive strips the Ethernet header first. Then it calls the interface
receive handler. The stack owner binds an interface to an `IpStack`; binding
installs `ip_receive` as that interface's receive handler and stores the stack
as the handler context.

```text
Ethernet frame
  -> ethernet_receive(iface, frame, &ethertype)
  -> iface->rx_handler(iface, frame, ethertype, iface->handler_ctx)
  -> ip_receive(iface, frame, ethertype, ip_stack)
```

At entry to `ip_receive`:

- `frame->data` points at the IPv4 header.
- `frame->len` is the IPv4 datagram length visible after Ethernet stripping.
- The old Ethernet header may still be readable in packet headroom at
  `frame->data - ETH_HDR_LEN`, if the packet was created with enough headroom.

Current `ip_receive` behavior:

1. Reject `NULL iface`, `NULL frame`, or `frame->len < IP_HDR_LEN`.
2. Interpret `frame->data` as `IpHeader *`.
3. Reject non-IPv4 packets.
4. Reject `ttl == 0`.
5. Reject bad header checksums. A valid complete header should make
   `ip_checksum(ip_hdr) == 0`.
6. Save `ip_hdr->protocol` before moving `frame->data`.
7. Add the original visible IPv4 length to `iface->rx_bytes`.
8. Interpret `ctx` as `IpStack *`.
9. Set `iface->last_rx_time` from `simulator_now(stack->sim)` when a stack and
   simulator are present; otherwise write `0`.
10. Advance `frame->data` by `IP_HDR_LEN`.
11. Decrease `frame->len` by `IP_HDR_LEN`.
12. Set `frame->layer = 4`.
13. If no stack is present, return `0` after stripping.
14. Look up the saved protocol number in the bound stack's protocol table.
15. If a handler exists, call it with the stripped packet and that handler's
    registered context.
16. If no handler exists, return `0` and leave protocol-unreachable behavior
    for a later IP/ICMP integration pass.

Layer-4 dispatch is generic. `ip.c` must not include `udp.h`, `tcp.h`, or
`icmp.h`, and it must not know about `UdpState`, `TcpTable`, or ICMP internals.
Upper protocols or a stack-owner module register handlers on an explicit
`IpStack`:

```c
typedef int (*IpProtocolHandler)(Interface *iface, Packet *pkt, void *ctx);

typedef struct IpStack IpStack;

void ip_stack_init(IpStack *stack, Simulator *sim);

int ip_stack_bind_interface(IpStack *stack, Interface *iface);

int ip_stack_register_protocol(IpStack *stack,
                               uint8_t protocol,
                               IpProtocolHandler handler,
                               void *ctx);

int ip_stack_unregister_protocol(IpStack *stack, uint8_t protocol);
```

Registration examples:

```c
IpStack ip_stack;
ip_stack_init(&ip_stack, sim);
ip_stack_bind_interface(&ip_stack, iface);

ip_stack_register_protocol(&ip_stack, IPPROTO_ICMP, icmp_receive, sim);
ip_stack_register_protocol(&ip_stack, IPPROTO_UDP, udp_receive, &udp_ctx);
```

This keeps the dependency direction correct:

```text
IP receive: validates IPv4 and demuxes by protocol number
ICMP/UDP/TCP: register receive handlers with an IpStack
UDP/TCP/ICMP send: call ip_output(...)
```

The registered handler receives a packet whose `data` points at the layer-4
header. The stripped IPv4 header remains readable at
`pkt->data - IP_HDR_LEN` when the packet came through normal IP receive.

Current failure behavior:

- Some malformed packets increment `rx_errors` or `rx_dropped` and free the
  packet.
- The null/too-short path returns `-1` without freeing the packet.

This asymmetry is current code behavior, not a final design recommendation.
When ownership is tightened later, all receive failure paths should be made
consistent.

## Send Boundaries

### `ip_output`

Use this from upper layers such as ICMP.

```c
int ip_output(Simulator *sim,
              uint32_t src_ip,
              uint32_t dst_ip,
              uint8_t protocol,
              Packet *payload);
```

Current behavior:

1. Reject `NULL sim` or `NULL payload`.
2. Search `sim->topo` for an interface whose `iface->ip_addr` equals
   `ns_htonl(src_ip)`.
3. Reject if no source interface is found.
4. Reject if the source interface is down.
5. Reject if the source interface has no `arp_cache`.
6. Convert the source interface IP to host order.
7. Check whether `dst_ip` is on the same subnet as the source interface using
   `iface->prefix_len`.
8. Reject non-local-subnet destinations. Route lookup is not implemented yet.
9. Look up `dst_ip` in the interface ARP cache.
10. On ARP hit, call
    `ip_send(sim, iface, dst_mac, src_ip, dst_ip, protocol, payload)`.
11. On ARP miss, send an ARP request for `dst_ip`, enqueue the payload through
    `arp_pending_enqueue`, and return `0` after the ARP cache pending queue
    accepts ownership of the payload.

This gives upper layers a clean API today while leaving the future expansion
point in the correct module.

Future `ip_output` behavior:

1. Otherwise perform longest-prefix route lookup for non-local destinations.
2. Use the route's next-hop IP, or the destination IP for connected routes.
3. Look up or resolve the next-hop MAC through ARP.
4. Forward packets through the selected outgoing interface.

### `ip_send`

Use this only when L2 resolution has already happened.

```c
int ip_send(Simulator *sim,
            Interface *iface,
            uint8_t dst_mac[6],
            uint32_t src_ip,
            uint32_t dst_ip,
            uint8_t protocol,
            Packet *payload);
```

Current behavior:

1. Reject `NULL sim`, `NULL iface`, `NULL dst_mac`, or `NULL payload`.
2. Build a stack `IpHeader`.
3. Set `version_ihl = (4 << 4) | (IP_HDR_LEN / 4)`.
4. Set `total_length = ns_htons(IP_HDR_LEN + payload->len)`.
5. Set `identification = 0`.
6. Set `flags_fragment_offset = 0`.
7. Set `ttl = IP_DEFAULT_TTL`.
8. Set `protocol` from the caller argument.
9. Set checksum to `0`, then compute and store `ip_checksum(&ip_hdr)`.
10. Convert `src_ip` and `dst_ip` to network byte order in the header.
11. Prepend the IPv4 header to `payload`.
12. Set `payload->layer = 3`.
13. Add the IPv4 packet length to `iface->tx_bytes`.
14. Set `iface->last_tx_time`.
15. Call `ethernet_send(sim, iface, dst_mac, ETHERTYPE_IPV4, payload)`.

On `packet_prepend` failure, `ip_send` returns `-1` and leaves Ethernet
unsent. Callers should treat payload ownership as still theirs on failure.

## Helper Functions

### `ip_checksum`

```c
uint16_t ip_checksum(IpHeader *ip_hdr);
```

Rules:

- `NULL` returns `0xFFFF`.
- The function reads the fixed 20-byte IPv4 header as ten 16-bit words.
- Each word is converted from network order to host order with `ns_ntohs`.
- Carries are folded back into the low 16 bits.
- The one's complement is returned in network byte order.
- For receive validation, a valid header including its checksum should produce
  `0`.

### `ip_is_local`

```c
int ip_is_local(Device *dev, uint32_t dst_ip);
```

Rules:

- `NULL dev` returns `0`.
- Returns `1` if any interface on `dev` has `iface->ip_addr == dst_ip`.
- Returns `0` otherwise.
- `dst_ip` must be in the same byte order as `iface->ip_addr`, which is network
  byte order.

## ACSL Contract Plan

The ACSL should stay source-truthful. Do not specify router dispatch or ARP
pending behavior until those paths exist.

Useful predicates:

```c
/*@ predicate ipv4_min_header_readable{L}(Packet *frame) =
      \valid(frame) &&
      frame->len >= IP_HDR_LEN &&
      \valid_read(frame->data + (0 .. IP_HDR_LEN - 1));

    predicate ipv4_checksum_ok{L}(Packet *frame) =
      ipv4_min_header_readable(frame) &&
      ip_checksum((IpHeader *)frame->data) == 0;
*/
```

Contract targets:

- `ip_init`: null simulator or null topology does nothing; valid topology
  assigns per-interface `rx_handler` and `handler_ctx`.
- `ip_receive`: separate null, too-short, bad-version, TTL-zero,
  bad-checksum, and valid-strip behaviors. Valid-strip ensures `frame->data`
  advances by `IP_HDR_LEN`, `frame->len` decreases by `IP_HDR_LEN`, and
  `frame->layer == 4`.
- `ip_output`: null inputs return `-1`; successful direct output means the
  payload eventually has both IPv4 and Ethernet headers prepended. The current
  implementation may return `-1` for no source interface, interface down,
  missing ARP cache, non-local destination, failed ARP request, or failed
  pending enqueue. An ARP miss with successful request and enqueue returns `0`
  because the ARP cache pending queue owns the payload until resolution.
- `ip_send`: null inputs return `-1`; prepend failure leaves the packet
  unsent; success adds an IPv4 header, sets `payload->layer = 3`, updates
  transmit counters, and calls Ethernet.
- `ip_checksum`: null returns `0xFFFF`; valid input does not mutate the
  header.
- `ip_is_local`: null device returns `0`; hit/miss behaviors are based on
  exact interface IP equality.

## Implementation Notes For The Next Step

The next clean network-stack milestone is not more ICMP logic. It is finishing
the IP output infrastructure:

1. Define a real route table entry and longest-prefix lookup API.
2. Give routers/hosts ownership of route tables and ARP caches.
3. Make `ip_output` choose direct destination vs route next hop.
4. Add IP receive dispatch to ICMP after the current strip/validate path.
5. Add forwarding path with TTL decrement and ICMP Time Exceeded.

After that, ICMP echo replies and ICMP errors can be implemented using
`ip_output(...)` without accepting destination MAC addresses in the ICMP public
API.
